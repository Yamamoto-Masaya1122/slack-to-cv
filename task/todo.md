## 📋 要件定義 最終版

### 前提・制約

| 項目 | 内容 |
|---|---|
| 動作環境 | ローカル（`localhost:3000`）のみ |
| 認証 | なし（Slack User Token を `.env` で管理） |
| データ保存 | DBなし・メッセージは保存しない |
| 技術スタック | Next.js 14 / TypeScript / Tailwind CSS / Zod |

---

### 画面構成

| 画面名 | パス | 内容 |
|---|---|---|
| 初期設定画面 | `/resume/register`（Step1） | 期間・チャンネル選択 |
| 職務経歴書作成画面 | `/resume/register`（Step2） | フォーム入力・ファイルダウンロード |

---

### 初期設定画面（Step1）

**フォーム**

| 項目 | UI |
|---|---|
| 対象期間（開始日） | DatePicker |
| 対象期間（終了日） | DatePicker |
| チャンネル選択 | 排他チェックボックス（1つ選んだら他を解除）＋ 検索フィルター |

**チャンネルデータ**
- ページ初期表示時に `GET /api/messages` で自動取得
- 表示：チャンネル名（`# dev-backend`）
- 送信：チャンネルID（`C0123456789`）

**バリデーション（ボタン押下時）**

| 項目 | ルール | エラーメッセージ |
|---|---|---|
| 開始日 | 必須 | 「開始日を入力してください」 |
| 終了日 | 必須 | 「終了日を入力してください」 |
| 期間の前後 | 開始日 ≤ 終了日 | 「終了日は開始日以降を指定してください」 |
| チャンネル | 1つ選択 | 「チャンネルを選択してください」 |

**ローディング（ボタン押下後）**

| 項目 | 内容 |
|---|---|
| 表示方式 | 画面全体を覆うオーバーレイ（最前面） |
| 背景 | 半透明の暗いオーバーレイ |
| 中央表示 | ローディングスピナー ＋「Slackからメッセージを取得中。」 |
| 操作制限 | オーバーレイ中は一切の操作を無効化 |
| 消えるタイミング | メッセージ取得完了後、Step2に切り替わると同時に非表示 |

### ダウンロードファイル仕様

**計2ファイルを同時ダウンロード**

**① `{channel_name}_messages.json`**

```json
{
  "channel": "dev-backend",
  "period": { "from": "2024-01-01", "to": "2024-12-31" },
  "messages": [
    {
      "type": "posted",
      "text": "DockerComposeの設定見直しました",
      "ts": "2024-03-01 10:00"
    }
  ]
}
```

**② `resume_draft.md`**

```markdown
# 職務経歴書生成指示

## あなたへのお願い
このファイルと同梱の `{channel_name}_messages.json` を元に、
「プロジェクト詳細・担当業務・主な実績」を生成してください。

## 生成ルール
以下のフォーマットを厳守してください。

▼担当業務
（担当した業務の概要・開発フェーズを1〜2文で）

▼詳細
■ Situation（状況）: どんなプロジェクト・チーム環境だったかを文章で
■ Task（課題）: どんな問題・課題があったかを文章で
■ Action（行動）: 課題に対して具体的に何をしたかを文章で
■ Result（結果）: 成果を文章で。可能な限り数値・割合・時間で定量表現

## 制約
- 全体400文字以内で出力すること
- Slackメッセージから読み取れない情報は根拠なく作らないこと
- 推測が必要な場合は合理的な範囲にとどめること
- 生成完了後はMarkdown形式でターミナルに表示すること
```

### ディレクトリ構成

```
app/
  resume/
    register/
      page.tsx            # 初期設定画面（Step1）+ 職務経歴書作成画面（Step2）

    messages/
      route.ts            # GET: チャンネル一覧 / POST: メッセージ抽出

  LoadingOverlay.tsx      # 全画面オーバーレイローディング
  TechTagInput.tsx        # 利用技術の自動補完タグ入力

  ChannelSelector.tsx     # チャンネル一覧・検索・排他チェックボックス

  MonthRangePicker.tsx    # 開始年月・終了年月の期間入力（Step2）
  ChannelForm.tsx         # 入力フォーム全体
constants/
  technologies.ts         # 利用技術定数
  validation.ts           # Zodスキーマ定義
lib/
  slack.ts                # Slack APIクライアント
types/
  index.ts                # 型定義
```

---

### コンポーネント設計方針

| 項目 | 内容 |
|---|---|
| 格納場所 | `components/` に全て集約 |
| 命名規則 | PascalCase（例: `ChannelSelector.tsx`） |
| 1ファイル1コンポーネント | 原則1ファイルにつき1つをexport |
| Props型定義 | 各ファイル内に `interface XxxProps` を定義 |
| 責務分離 | データ取得・状態管理は `page.tsx` に集約、コンポーネントはUI描画に専念 |
